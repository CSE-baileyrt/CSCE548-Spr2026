DOM XSS — How it Differs

DOM-based XSS never reaches the server. Client-side JavaScript reads from location.hash, location.search, or document.referrer and writes it unsafely to the DOM.

The htmlEncode fix — Java

// VULNERABLE — raw output out.println("Search: " + searchTerm); // SECURE — HTML-encode before output String safe = htmlEncode(searchTerm); out.println("Search: " + safe); // The encoder private static String htmlEncode(String s) { return s .replace("&", "&") .replace("<", "<") .replace(">", ">") .replace("\"", """) .replace("'", "'"); } // Better: use a library // OWASP Java Encoder: // Encode.forHtml(searchTerm)
